üìä Table Types

üì¶

Table (Physical Table)

Created via CREATE TABLE AS

Persistent storage, can be referenced by subsequent queries

CREATE TABLE dept_stats AS
SELECT dept_id, AVG(salary) FROM employees GROUP BY dept_id;
üëÅÔ∏è

View (Virtual Table)

Created via CREATE VIEW

No data storage, computed dynamically on query

CREATE VIEW high_salary_employees AS
SELECT * FROM employees WHERE salary > 50000;
üîÑ

CTE (Common Table Expression)

Defined in WITH clause

Temporary named result set, valid only within current query, can be recursive

WITH monthly_sales AS (
    SELECT dept_id, SUM(amount) as total
    FROM sales GROUP BY dept_id
)
SELECT * FROM monthly_sales;
üìä

Subquery (Derived Table)

Nested inline query

Temporary result, must have alias

SELECT * FROM (
    SELECT dept_id, COUNT(*) as cnt
    FROM employees GROUP BY dept_id
) AS dept_counts;
üåê

External (Source Table)

Referenced but not defined in current SQL

Data source, typically the starting point of lineage

Example: employees, orders tables

üìù Example Analysis

Try this SQL:

WITH sales_summary AS (
    SELECT product_id, SUM(amount) as total_sales
    FROM sales
    GROUP BY product_id
)
SELECT 
    p.name as product_name,
    ss.total_sales
FROM products p
JOIN sales_summary ss ON p.id = ss.product_id
WHERE ss.total_sales > 1000;

This will show: CTE (yellow), External tables (gray), Aggregate fields (pink), and complete lineage paths.
